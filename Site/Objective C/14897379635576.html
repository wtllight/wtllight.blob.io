内存管理

  1. 堆栈heap,stack的区别

    • 栈区stack

      1. 内存由编译器自动分配释放
      2. 是一块连续的内存区域
      3. 是向低地址扩展的数据结构
      4. 栈顶的地址和栈的最大内容是系统预先规定好的
      5. 栈是先进后出的队列，不存在内存块从中间弹出的现象
      6. 内容包括，方法参数值，局部变量值等

    • 堆区 heap

      1. 一般由程序员分配释放，如不释放，APP Close时，OS回收
      2. 向高地址扩展的数据结构
      3. 内存区域不连续，获得的空间较灵活

    • 内存碎片

      1. 堆内存，new/delete,会造成内存空间的不连续
      2. 栈空间不存在碎片的问题

    • 分配方式

      1. 堆是动态分配的，没有静态分配的堆
      2. 栈有静态分配与动态分配两种方式，他的释放由编译器完成
      3. 静态分配由编译器完成
      4. 动态分配是有alloc函数进行分配

    • 分配效率

      1. 栈是机器系统提供的数据结构，计算机底层提供支持
      2. 分配专门的寄存器存放栈的地址
      3. 压栈，出栈有专门的指令执行
      4. 对的内存分配是C/C++函数提供，机制比较复杂

    • 全局变量(static)

      1. 全局与静态变量是放在一块的，未初始化的全局与static在一块
      2. 初始化的全局变量与static 在相邻的另一块区域
      3. 常量区，常量字符串存放区域
      4. 代码区，函数体二进制代码

  2. 浅拷贝与深拷贝

    1. 指针只是存储内存地址中资源地址的占位符
    2. 如果只是复制指针，底层资源实际仍然是两个实例在共享
    3. 浅拷贝，指针仍然指向内存中同一资源
    4. 深拷贝，不仅复制指针值，而且复制指针指向的资源

  3. IOS的内存管理

    1. Objective C中的对象，都是基于引用计数来管理生命周期的
    2. 当一个对象的引用计数为0时，这个对象会被自动销毁
    3. MRC，手动管理对象的引用计数，明确控制对象的生命周期
    4. MRC，接口对引用计数的处理，对象放进数组，引用计数会加1，使用alloc创建的对象，引用计数一开始就是1，有些接口创建的对象一开始就被调用过autorelease
    5. ARC，是编译器通过对代码的静态分析，确定对象的生命周期，并在合适的位置加上retain与release。

  4. ARC

    1. AutoReferenceCounting(ARC),编译器来进行内存管理
    2. 对象创建时，count + 1，，release 时 - 1，count = 0 时，销毁
    3. autoreleasepool时，自动加上autorelease，count = 0时，销毁
    4. 堆内存释放时，要确定指向该空间所有的指针都被release
    5. 要确定，哪些指针指向同一个堆内存，这些指针只能释放一次
    6. MRC下，是谁创建，谁释放
    7. 模块化操作时，对象可能被多个模块创建和使用
    8. 多线程操作时，不确定那个线程最后使用完毕

  5. 深入ARC

    1. 不仅编译时自动添加retain，release,autorelease
    2. 不能手动进行代码的内存管控
    3. dealloc虽然可以被调用，但是不能调用[super dealloc]
    4. 不能使用new开始属性名称
    5. ARC也包括运行期的组件

  6. objc对象的内存布局

    1. 所有父类的成员变量和自己的成员变量，都会存在该对象对应的存储空间
    2. 每一个对象内部都有一个isa指针，指向他的类对象
    3. 类对象中存在着该对象的方法列表，成员变量列表，属性列表，和一个指向metal class(元对象)的isa指针
    4. 元对象内部存放着，类方法列表，还有一个superclass的指针，指向他的父类元对象

  7. Autorelease的释放

    1. 手动干预释放，指定autoreleasepool，该pool 结束时释放
    2. 系统自动释放，当前runloop结束时释放
    3. ViewDidLoad结束时添加autorelease，那么viewDidAppear执行前就被销毁了
    4. 一个thread中使用autorelease，该thread结束时，被销毁

  8. Autorelease的实现

    1. autoreleasepool 以一个栈的形式实现
    2. objc_autoreleasepoolPush
    3. objc_autoreleasepoolPop
    4. objc_autorelease
    5. 即对autorelease进行压栈，出栈操作，销毁时，进行releease操作
    6. retain，block中容易出现retain cycle，故用到self的地方使用__weak,弱引用
