多线程

  1. 线程与进程的区别

    1. 一个程序至少要有一个进程，一个进程至少要有一个线程
    2. 进程，资源分配的最小独立单元，由系统进行资源分配与调度
    3. 线程，是进程下的一个分支，是进程的实体，是CPU调度和分配的最小单元
    4. 线程是比进程更小的独立运行的基本单位，可以与同一进程的其他线程共享系统资源
    5. 线程只是一个进程中不同的执行路径，线程有自己的堆栈和局部变量。
    6. 一个线程死掉，基本会引起一个进程的死掉，所以多进程要比多线程程序健壮
    7. 但是对于某些同时进行，并且要共享某些变量的操作，只能使用线程来操作

  2. 多线程

    1. 同步完成多项任务，CPU：多核，操作系统:多任务，应用：多操作，IOS中：并发
    2. IOS，Cocoa可以创建线程化的对象
    3. GCD(Grand Central Dispatch),系统多线程类库，C语言，多核心，Block
    4. NSOperation封装了线程的细节实现，与NSQueue一起管理多线程
    5. NSThread，控制线程执行的对象，NSThread线程之间的并发控制，可以通过NSCondition来实现

  3. NSCondition

    1. NSCondition 是对线程的高度抽象
    2. 提供了GCD不是很容易复制的有用特性
    3. 提供了任务的状态，可以方便的取消一个NSOperation的运行
    4. GCD，简单易用，节省代码，block的使用，通俗易懂，可在简单项目使用

  4. 队列与多线程

    1. 串行队列，队列中的任务顺序执行

      dispatch_queue_t q = dispatch_queue_create("",DISPATCH_QUEUE_SERIAL)

      1. 并行队列，队列中的任务通常会并发执行

      dispatch_queue_t q = dispatch_queue_create("",DISPATCH_QUEUE_CONCURRENT)

      1. 全局队列，系统的，可以直接使用，与并行队列类似

      dispatch_queue_t q = dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT,0)

      1. 主队列 dispatch_queue_t q = dispatch_get_main_queue()

    2. atomic与线程安全

      1. atomic属性原子性，是指属性的存取方法是线程安全的，并不能保证整个对象是线程安全的
      2. 该属性的其他方法(except setter,getter)并不是线程安全的，需要用lock来保证其线程安全性

    3. 线程的创建

      1. 三种方法，GCD的dispatch，NSThread，NSOperation与NSOperationQueue
      2. 主线程执行方法performSelectorOnMainThread
      3. 延迟执行代码 “performSelector:onThread:withObject:waitUntilDone”
