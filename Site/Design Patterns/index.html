Chapter 1 ： Hello Design Patterns

  1. 准备

    1. MAC 电脑与 Xcode
2. 线框图  www.omnigroup.com/products/omnigraffle/与UI布局
3. 类与对象图，建模图示法
4. 专业丛书而非基础读物
5. 解决似曾相识的问题，特定场景下类与对象的通信
6. 优先使用组合而非继承，针对接口而非实现编程

  2. 模式起源：MVC

    1. MVC，对象在APP中的三组分类
2. MVC,对象之间跨越抽象边界的通信方式
3. MVC，不是独立的模式，而是其他基本模式组成的复合模式
4. MVC模式，对象更容易扩展与复用，接口的便利性
5. 技术与架构建立在MVC角色设定之上，要求类与对象服从角色设定的规则
6. Model Object === Data struct
7. View Object == Display && User Action --All Model 
8. Controller Object == Agent && Coordinate --Message

  3. 用到的基本设计模式：组合，命令，中介者，策略，观察者

    1. Composite, View --> View 
2. Command, View -->Controller
3. Mediator, Model<Controller>View
4. Strategy, Controller -->View
5. Observer, Model-->Controller

  4. 设计的几个问题

    1. 针对接口编程，而不是针对实现编程

        1. 类， 对象，继承，多态，接口
        2. 类继承与接口继承
        3. 类继承用于复用父类的功能，来定义对象的实现，即继承功能
        4. 接口定义了类型，接口继承可以让我们用一个对象代替另一个
        5. 多态是基于接口的，具体为Protocel

2. @protocel与抽象基类

        1. 协议，定义了若干抽象行为的接口
        2. 抽象基类，定义了预留了应该由子类重载的行为
        3. 协议解决的是类无法多重继承的问题

3. 对象组合与类继承

        1. 对象组合与类继承都可以使用复用的方法来定义实现。
        2. 继承存在实现的依存关系
        3. 对象组合可以让我们使用多个对象

  5. 本书用到的对象与类

    1. 图形与图表来解释模式对象的结构，对象图
2. 类图用来说明类与类之间的静态关系和类的结构
3. 类图表示：协议，抽象类，具体类，范畴的图示
4. 关系：实例化，继承，子类型化，相识，聚合，伪代码
5. 对象图
